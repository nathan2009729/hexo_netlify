HackerRank-Lower Bound-STL

2019-04-13
  • HackerRank OJ
  • STL
  • Vector
  • binary_search
  • c++基礎語法練習
  • lower_bound

Problem

問題連結

Solution

這題是拿來練習C++中的STL中的binary_search跟lower_bound函數，兩個函數皆在頭文件algorithm內。binary_search是以二分搜索演算法尋找所需要的數字是否在vector內，lower_bound則是以二分搜索演算法找到元素後，回傳那個數值的位置，如果找不到，則會回傳第一個大於它的值的位置。要注意，如果想找的數字比vector內任何一個數字要大，則會回傳.end()，這是越界的。

The aim of this problem is to practice function binary_search and lower_bound of STL in C++. The binary_search() algorithm implements a binary search, it searches the range specified by the first two arguments for an element that is equivalent to the third argument. This algorithm return a bool value that it is true if the third argument is found, or false otherwise. So binary_search() algorithm only tells you whether or not the element is present, but not where it is when it is present. The lower_bound() algorithm finds element in the range specified by the first two arguments that is not less than the third arguments, and returns an iterator pointing to the first element in the range specified by the first two arguments which has the value not less than the third argument.

HackerRank-Structuring the Document

2019-04-06
  • HackerRank OJ
  • Pointer
  • Struct
  • 基礎語法練習

Problem

問題連結

Solution

這題要考的是對C裡struct的了解，blog主搞了一個禮拜。

這題的題意是給予一個文件(document)，文件含有數個paragraph，paragraph由一至數個sentence組成，一個sentence由一個以上word組成，word由character組成。不同的paragraph由分行號(newline，\n)區隔，不同的sentence由句點(.)區隔，不同的word由空白區隔。

先說說如何宣告變數。因為每次輸入只會有一個文件，所以宣告文件時僅需struct document doc;即可。但其他的paragraph、sentence、word等，根本不知道具體的數量(只知道最多5個paragraph、1000個characters)，怎麼辦? 只好用動態分配。

但是struct的宣告又是一個包一個的，像是

文件struct又包了paragraph的struct。所以，要宣告名為doc的文件含5個paragraph要怎麼辦? 答案是
doc.data = malloc(5*sizeof(struct paragraph));
第i個paragraph含有100個sentence要怎麼辦?
doc.data[i].data = malloc(100 * sizeof(struct sentence));
其他的就是以此類推。

再來說說存取的部分。如果我們要存取第1個paragraph的第2個sentence的第3個word的第4個字母(character)，怎麼辦呢?
就是doc.data[1].data[2].data[3].data[4]。

原本blog主是想到doc->data這樣的寫法，但這樣有問題: 如果要存取第三個paragraph的話呢?會變成(doc->data)[3]，要存取第三個paragraph的第i個word呢?會變成很不直觀、不好理解的寫法。而且，一開始要聲明的時候，不是宣告成struct document doc而是struct document doc。
這是因為， ->(arrow) operator只是syntax sugar，a->b其實是(\a).b的簡易寫法罷了。所以doc->data就是(*doc).data，第3個paragraph就是(((\doc).data)+3)。又比如，如果不只一個文件，存取第2個文件的第3個paragraph就是*(*(doc+1).data)+3。
我去你的，很複雜阿。因此->寫法似乎比較適合paragraph、sentence等等都只有1個的狀況，如果多個paragraph、sentence，還是不要耍炫，乖乖用.吧。

其餘，是發現一些奇怪的問題。第59行的wr，其實就是代表word_count，但在迴圈結束前把它的值assign給doc.data[i].data[j].word_count卻會顯示0。另外，如果sentence、word、character都各配置1000個(也就是有1000個sentence，每個sentence有1000個word，每個word有1000個字)，則會segmentation fault。

Aizu IPT1_11_C Dice III

2019-03-23
  • Aizu OJ

Problem

https://onlinejudge.u-aizu.ac.jp/courses/lesson/2/ITP1/11/ITP1_11_C

Dice III

Write a program which reads the two dices constructed in the same way as Dice I, and determines whether these two dices are identical. You can roll a dice in the same way as Dice I, and if all integers observed from the six directions are the same as that of another dice, these dices can be considered as identical.

Input

In the first line, six integers assigned to faces of a dice are given in ascending order of their corresponding labels.
In the second line, six integers assigned to faces of another dice are given in ascending order of their corresponding labels.

Output

Print “Yes” if two dices are identical, otherwise “No” in a line.

Constraints

0≤ the integer assigned to a face ≤100

Sample Input

Sample Input 1

1 2 3 4 5 6
6 2 4 3 5 1

Sample Input 2

1 2 3 4 5 6
6 5 4 3 2 1

Sample Output

Sample Output 1

Yes

Sample Output 2

No

Solution

題意是給出兩種點數排列，判斷是否是同一個骰子。當然兩行不會一模一樣，只要其中一個骰子經過旋轉可跟第二個一樣即可。這題本來也想的很複雜，像是判斷每一面的對面是否位置加起來是七，以及透過頂方跟前方，來判斷右方是否合理。但其實不用，只要把其中一個骰子用盡所有骰法，有一種跟第二列一樣就OK了。骰法只有24種，頂部可能為1-6其中一種(由line 64的i控制)，確定頂部後，往逆時針轉90度4次(由line 71的j控制)，所以6*4=24。什麼是逆時針? 把骰子先往東、再往南、再往西，就是逆時針90度。

Problem

https://onlinejudge.u-aizu.ac.jp/courses/lesson/2/ITP1/11/ITP1_11_B

Dice II

Construct a dice from a given sequence of integers in the same way as Dice I.

You are given integers on the top face and the front face after the dice was rolled in the same way as Dice I. Write a program to print an integer on the right side face.

Input

In the first line, six integers assigned to faces are given in ascending order of their corresponding labels. In the second line, the number of questions q is given.

In the following q lines, q questions are given. Each question consists of two integers on the top face and the front face respectively.

Output

For each question, print the integer on the right side face.

Constraints

0≤the integer assigned to a face≤100
The integers are all different
1≤q≤24

Sample Input

Sample Input
1 2 3 4 5 6
3
6 5
1 3
3 2

Sample Output

Sample Output
3
5
6

Solution

這題沒什麼好說，就是把所有可能直接寫死。right[up][front]代表頂部值為up，前方值為front時，右方的值為何。要注意一個小技巧，up跟front代表的是哪一面，right[up][front]代表哪一個點數。因為輸入up跟front時是輸入點數，所以要找出該點數是哪一面，對應程式碼就是

if(up == dice_num[j]) up = j;

if(front == dice_num[j]) front = j;這兩行

Problem

https://onlinejudge.u-aizu.ac.jp/courses/lesson/2/ITP1/11/ITP1_11_A

Dice I

Write a program to simulate rolling a dice, which can be constructed by the following net.

As shown in the figures, each face is identified by a different label from 1 to 6.

Write a program which reads integers assigned to each face identified by the label and a sequence of commands to roll the dice, and prints the integer on the top face. At the initial state, the dice is located as shown in the above figures.

Input

In the first line, six integers assigned to faces are given in ascending order of their corresponding labels.

In the second line, a string which represents a sequence of commands, is given. The command is one of ‘E’, ‘N’, ‘S’ and ‘W’ representing four directions shown in the above figures.

Output

Print the integer which appears on the top face after the simulation.

Constraints

0≤ the integer assigned to a face ≤100
0≤ the length of the command ≤100

Sample Input

Sample Input 1

1 2 4 8 16 32
SE

Sample Input 2

1 2 4 8 16 32
EESWN

Sample Output

Sample Output 1

8
You are given a dice where 1, 2, 4, 8, 16 are 32 are assigned to a face labeled by 1, 2, …, 6 respectively. After you roll the dice to the direction S and then to the direction E, you can see 8 on the top face.

Sample Output 2

32

Solution

這題本來想得很複雜，以為要記住所有翻轉的可能，像是頂部為1前方為2時轉東南西北會是什麼。後來發現沒有必要，只要實際轉轉骰子，記錄哪一面被哪一面取代就好。比如說，

1 2 3 4 5 6往北轉，會發現點數1的面被點數2取代，那就dice[1] = dice[2]，其他以此類推。
