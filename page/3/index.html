Aizu ALDS1 3-B Queue

Problem

https://onlinejudge.u-aizu.ac.jp/courses/lesson/1/ALDS1/3/ALDS1_3_B

Queue

There are n processes in a queue. Each process has namei and timei. The round-robin scheduling handles the processes in order. A round-robin scheduler gives each process a quantum (a time slot) and interrupts the process if it is not completed by then. The process is resumed and moved to the end of the queue, then the scheduler handles the next process in the queue.

For example, we have the following queue with the quantum of 100ms.

A(150) - B(80) - C(200) - D(200)
First, process A is handled for 100ms, then the process is moved to the end of the queue with the remaining time (50ms).

B(80) - C(200) - D(200) - A(50)
Next, process B is handled for 80ms. The process is completed with the time stamp of 180ms and removed from the queue.

C(200) - D(200) - A(50)
Your task is to write a program which simulates the round-robin scheduling.

Input

n q
name1 time1
name2 time2
…
name_n time_n
In the first line the number of processes n and the quantum q are given separated by a single space.

In the following n lines, names and times for the n processes are given. namei and timei are separated by a single space.

Output

For each process, prints its name and the time the process finished in order.

Constraints

1 ≤ n ≤ 100000
1 ≤ q ≤ 1000
1 ≤ timei ≤ 50000
1 ≤ length of namei ≤ 10
1 ≤ Sum of timei ≤ 1000000

Sample Input

5 100
p1 150
p2 80
p3 200
p4 350
p5 20

Sample Output

p2 180
p5 400
p1 450
p3 550
p4 800

Stack

Reverse Polish notation is a notation where every operator follows all of its operands. For example, an expression (1+2)*(5+4) in the conventional Polish notation can be represented as 1 2 + 5 4 + * in the Reverse Polish notation. One of advantages of the Reverse Polish notation is that it is parenthesis-free.

Write a program which reads an expression in the Reverse Polish notation and prints the computational result.

An expression in the Reverse Polish notation is calculated using a stack. To evaluate the expression, the program should read symbols in order. If the symbol is an operand, the corresponding value should be pushed into the stack. On the other hand, if the symbols is an operator, the program should pop two elements from the stack, perform the corresponding operations, then push the result in to the stack. The program should repeat this operations.

Input

An expression is given in a line. Two consequtive symbols (operand or operator) are separated by a space character.

You can assume that +, - and * are given as the operator and an operand is a positive integer less than $10^6$

Output

Print the computational result in a line.

Constraints

  • $2 \leq$ the number of operands in the expression $\leq 200,000$
  • $-10^9 \leq$ values in the stack $\leq 10^9$

Sample Input 1

Sample Input 2

Maximum Profit

You can obtain profits from foreign exchange margin transactions. For example, if you buy 1000 dollar at a rate of 100 yen per dollar, and sell them at a rate of 108 yen per dollar, you can obtain (108 - 100) × 1000 = 8000 yen.

Write a program which reads values of a currency $R_t$ at a certain time $t$($t = 0,1,2,\ldots n-1$), and reports the maximum value of $R_j-R_i$ where $j > i$.

Input

The first line contains an integer $n$. In the following $n$ lines, $R_t(t = 0,1,2,\ldots n-1$) are given in order.

Output

Print the maximum value in a line.

Constraints

  • $2 \leq n \leq 200,000$
  • $1 \leq R_t \leq 10^9$

Sample Input 1

Sample Input 2

4

Sample Output

Sample Output 1

Solution

This problem can be solved by using divide and conquer technique. We split the array in half. The maximum profit can be founded in one of the following three situations:

  1. minimum value and maximum value in the first half of array(the buy/sell pair occurs completely within the first half)
  2. minimum value and maximum value in the second half of array(the buy/sell pair occurs completely within the second half)
  3. minimum value in the first half of array, maximum value in the second half of array(the buy/sell pair occurs across both halves-we buy in the first half, then sell in the second half.)

we can get values for 1 and 2 by recursively invoking our algorithms on the first and second halves.
For option 3, we have to find minimum value in the first half, maximum value in the second half. we can do it by doing a simple linear scan, or store the values of minimum value and maximun value in subproblem.

Halting Problem-using diagonalization method

定義1:(d1)
(<M>,x)為機器<M>輸入x時的輸出。輸出有三種情形，一是輸出非0且停機、二是輸出0且停機、三是無限循環也就是不停機。
定義2:(d2)
halting function h
h(<M>,x)=1 $\Leftrightarrow$ (<M>,x)停機，
Theorem 1.5:
The halting function h is not computable.
Proof:
要先知道，機器(也可以想成演算法)有很多種的，像是下圖，有機器T1、T2…等等。這是因為我們想知道世界上所有的演算法是不是有一個可以解決停機問題。下圖的表格，就是演算法Ti輸入演算法Tj時會發生什麼事，是停機呢(輸出Yes或No)，還是進入無限循環。

此圖引用於http://www.regor.url.tw/cnchou/2016/06/14/〈計算理論很好玩〉計算模型23：不可決定性問題/

而證明的方法，是證明即使將h限定到其對角線上也不可解。對角線也就是機器自己輸入自己，我們定義h限定到其對角線上這個函數為d，具體來說，d(<M>) = h(<M>,<M>)。
定義3:(d3)
d(<M>) = h(<M>,<M>)

我們會先構造一個和d相關的函數d’，先證明d’不可計算。d如果可以計算，那d’也很好算; 但如果d’不可計算，d就不能計算了。又因為d不可計算，clearly，h()也不可計算。

Lemma1(L1): 證明d’不可計算:
先考慮一個與d相關的函數d’，這函數是被定義為做反事的(The function d’ is defined on purpose so to foil any attempt to compute it)，即對任意機器M，d’(<M>)的值不同於(<M>,<M>)。

定義4:(d4)
d’(<M>) = 1 $\Leftrightarrow$ M halt on input M with output 0
(That is, 在無限循環或輸出不為0時, d’(<M>) = 0)

現在假設有一機器$M_{d’}$可計算d’且對任何輸入停機。由d4，d’(<$M_{d’}$>) = 1$\Leftrightarrow$(<$M_{d’}$>,<$M_{d’}$>) = 0.也就是說，因為$M_{d’}$是用來計算d’，所以(<$M_{d’}$>,<$M_{d’}$>)跟d’(<$M_{d’}$>)應該要一樣的值，但照定義卻不同，因此有矛盾。
Lemma1 QED

Lemma2(L2): 利用d來算出d’的algo:
再來，構造兩個機器M’跟M’’，使得
d(<M’’>) = d’(<M’>)，也就是說，如果d是可計算的，那也可以算出d’。但L1已證明d’不可計算，所以d也無法計算。
M’’ is defined to operates as follows:

  1. on input x, M’’ emulates M’ on x.
  2. If M’ halt on input x with output 0, M’’ halts.
  3. If M’ halt on input x with an output different from 0, then M’’ infimite loop(不停機)
  4. Otherwise, M’’ does not halt.

在步驟2，d’(<M’>) = 1(by d4)
d(<M’’>) = h(<M’’>,<M’’>)(by d3)，
h(<M’’>,<M’’>) = 1(by d2)
$\Rightarrow$ d(<M’’>) = 1
所以步驟2中d(<M’’>) = d’(<M’>)，步驟三跟四也符合這等式。
Lemma2(L2) QED

根據L2，我們可以利用d來算出d’。試著想像一下，有一個算d’的演算法A調用了算出d的演算法B，如果d’根本不能算，那代表演算法B也是不能算的，也就是d也算不出來。

Problem

Solution

這題是拿來練習C++中的STL中的binary_search跟lower_bound函數，兩個函數皆在頭文件algorithm內。binary_search是以二分搜索演算法尋找所需要的數字是否在vector內，lower_bound則是以二分搜索演算法找到元素後，回傳那個數值的位置，如果找不到，則會回傳第一個大於它的值的位置。要注意，如果想找的數字比vector內任何一個數字要大，則會回傳.end()，這是越界的。

The aim of this problem is to practice function binary_search and lower_bound of STL in C++. The binary_search() algorithm implements a binary search, it searches the range specified by the first two arguments for an element that is equivalent to the third argument. This algorithm return a bool value that it is true if the third argument is found, or false otherwise. So binary_search() algorithm only tells you whether or not the element is present, but not where it is when it is present. The lower_bound() algorithm finds element in the range specified by the first two arguments that is not less than the third arguments, and returns an iterator pointing to the first element in the range specified by the first two arguments which has the value not less than the third argument.
