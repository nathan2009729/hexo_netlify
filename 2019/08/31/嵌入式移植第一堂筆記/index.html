嵌入式移植第一堂筆記

  • Embedded Linux系統移植實作

整備本機環境

使用的環境是ubuntu，先打開終端機，執行以下指令，準備好等下弄板子需要的程式工具。

  1. sudo apt-get update
  2. sudo apt-get install git gnupg flex bison gperf build-essential zip curl libc6-dev libncurses5-dev x11proto-core-dev libx11-dev:i386 libreadline6-dev:i386 libgl1-mesa-glx:i386 libgl1-mesa-dev g++-multilib tofrodos python-markdown libxml2-utils xsltproc zlib1g-dev:i386
  3. sudo ln -s /usr/lib/i386-linux-gnu/mesa/libGL.so.1 /usr/lib/i386-linux-gnu/libGL.so

  4. 修改文件，以下載mingw32。這是因為自ubuntu16起，mingw32從台灣載點被刪除，我們要加入美國的伺服器位址才能載的回來。
    打開需修改的文件(/etc/apt/sources.list)
    sudo gedit /etc/apt/sources.list
    在裡面加上
    deb http://us.archive.ubuntu.com/ubuntu trusty main universe
    儲存後離開。

  5. sudo apt-get update

  6. sudo apt-get install mingw32
  7. 安裝filezilla
    sudo apt‐get install filezilla
  8. 安裝minicom，要透過這程式來設定連接板子的usb。
    sudo apt-get install minicom

連到板子

用的板子是cortex系列。會用到兩條線:
usb-to-rs232(型號是u232-p9)
vga轉接(是母對母的)

可以看到u232-p9的一端是usb另一端是vga(公)。把usb端接到電腦，vga(公)接到vga轉接線，再把vga轉接線接到板子上的vga接頭(公)。這時打指令lsmod可看到下面一行:
usbserial 49152 3 pl2303
代表這線是pl2303的driver。也有人是另一條不同driver的線(ch43x)。

再打ls /dev/tty*這指令，檢查一下電腦是否有抓到u232-p9，如果有，應該會多一個/dev/ttyUSB0。

整備編譯環境

[以下皆在電腦端執行]
把提供的arm-4.5.1.tar.bz2給解壓縮到/usr/local資料夾內，這個資料夾內有arm平台的編譯工具(像是arm-linux-gcc，這是gcc的arm版)。
sudo tar -jxvf arm-4.5.1.tar.bz2 -C /usr/local
解壓縮結束後，如果每次要用到arm編譯工具都要先cd到/usr/local再使用，但這太不方便，所以要把arm編譯工具的路徑加入環境變數，這樣就不用每次都先cd到/usr/local。
加入環境變數步驟如下:

  1. sudo vim ~/.bashrc
  2. 開啟bashrc檔案，並再最後一行加入
    export PATH=/usr/local/arm/4.5.1/bin:$PATH
  3. 再執行
    source ~/.bashrc
    即可。

在hello.c所在的資料夾(哪都沒關係)，執行以下指令
arm-linux-gcc hello.c -o hello
如果生成名為hello的執行檔，即成功。注意在電腦上是無法執行的，只能在arm平台上執行。

燒錄準備

這款cortex A9用的處理器是三星Exynos 4412，我們要燒錄開機程式至板子上，需要灌一些燒錄用的工具。
(以下都在電腦端)首先下載dnw.tar.bz2並解壓縮
sudo tar ‐jxvf dnw.tar.bz2

解壓縮後出現dnw資料夾，我們須編譯裡面的dnw_new.c
輸入gcc dnw_new.c -o dnw會出現錯誤:

這是因為我們沒裝libusb-dev，因此下以下指令安裝
sudo apt-get install libusb-dev
安裝libusb-dev完後，再編譯一次依然有錯:

出現的是undefined reference，這是因為編譯器不知道用到的函數要去哪裡找，所以要打以下指令才會編譯成功:
gcc dnw_new.c -o dnw -lusb

接下來試試看如何傳資料:

  1. 在板子端開啟Minicom，輸入dnw。
  2. 在電腦端隨便傳一個檔案，假設是note.txt
    sudo dnw E4412_N.bl1.bin

    板子端也會出現傳輸成功。

Bootloader傳輸實驗

  1. [板子端]emmc open 0
  2. [板子端]dnw
  3. [電腦端] sudo dnw E4412_N.bl1.bin

    板子端會出現以下畫面:
  4. [板子端]mmc write 0 0xc0000000 0 0x10

    把E4412_N.bl1.bin寫入0x10
    以下指令不貼圖，因為只是傳不同的檔案，畫面類似。
  5. [板子端]dnw
  6. [電腦端]sudo dnw bl2.bin
  7. [板子端]mmc write 0 0xc0000000 0x10 0x1C
    再來一組，
  8. [板子端]dnw
  9. [電腦端]sudo dnw u-boot.bin
  10. [板子端] mmc write 0 0xc0000000 0x30 0x21D
    最後關閉，再重啟
  11. [板子端]emmc close 0
  12. [板子端]reset

FastBoot

  1. [電腦端]chmod 777 fastboot
  2. [電腦端]sudo cp fastboot /sbin
  3. [板子端]fastboot
  4. [電腦端]sudo fastboot flash kernel zImage

  5. [板子端]reset

    如果再次restart板子，進入minicom後輸入boot指令，可看到有四隻企鵝在板子螢幕上。

    安裝nfs (network file system)

    首先就是安裝相關套件:

  6. sudo apt-get install nfs-common nfs-kernel-server
    再來下載課堂提供的root_mkfs.tar.bz2，並解壓縮:
  7. sudo tar -jxvf root_mkfs.tar.bz2
    解壓縮後用指令pwd，會顯示root_mkfs的路徑，等等會用到
  8. pwd
    假設pwd後顯示出來的路徑是/home/cadtc/root_mkfs。
    nfs-common跟nfs-kernel-server安裝完成後etc目錄內會出現exports檔案，要在最後一行加上剛剛顯示的root_mkfs的路徑
  9. sudo vim /etc/exports
    執行上述命令後，進入exports檔案，最後一行加上
    /home/cadtc/root_mkfs *(rw,sync,no_root_squash)
    儲存後離開。
  10. sudo /etc/init.d/nfs-kernel-server restart
    出現如上圖最後一行的ok即成功。
完
