移植音樂播放器

移植 libid3tag:
./configure —host=arm-linux CC=arm-linux-gcc —prefix=/usr/local/arm/4.5.1/arm-none-linux-gnueabi
make -j8
sudo -s
make install

移植libmad:
./configure —host=arm-linux CC=arm-linux-gcc —prefix=/usr/local/arm/4.5.1/arm-none-linux-gnueabi
make -j8
make install

在make會出現 make error message 

arm-linux-gcc -DHAVE_CONFIG_H -I. -I. -I. -DFPM_ARM -DASO_INTERLEAVE1 -DASO_IMDCT -Wall -g -O -fforce-mem -fforce-addr -fthread-jumps -fcse-follow-jumps -fcse-skip-blocks -fexpensive-optimizations -fregmove -fschedule-insns2 -fstrength-reduce -MT version.lo -MD -MP -MF .deps/version.Tpo -c version.c  -fPIC -DPIC -o .libs/version.o
c1: error: unrecognized command line option "-fforce-mem"
make[2]: *** [version.lo] Error 1
make[2]: Leaving directory `/home/cadtc/libmad-0.15.1b'
make[1]: *** [all-recursive] Error 1
make[1]: Leaving directory `/home/cadtc/libmad-0.15.1b'
make: *** [all] Error 2
===============

只要把Makefile中所有的-fforce-mem通通刪除，再重新make即可

編譯madplayer:
./configure —host=arm-linux CC=arm-linux-gcc
make -j8

移動library
cp -rf /usr/local/arm/4.5.1/arm-none-linux-gnueabi/lib/libid3tag. ~/root_mkfs/lib
cp -rf /usr/local/arm/4.5.1/arm-none-linux-gnueabi/lib/libmad. ~/root_mkfs/lib
把madplay移到板子端
cp ~/mp3player/madplay-0.15.0b/madplay ~/root_mkfs

但是做完以上，執行./madplayer xxx.mp3時依然沒聲音，這是因為沒有掛載音效卡驅動。
複製音效卡驅動(snd-soc-wm8960.ko跟snd-soc-tiny4412-wm8960.ko)至板子端
cp snd-soc-wm8960.ko snd-soc-tiny4412-wm8960.ko ~/root_mkfs/lib/modules/3.5.0-FriendlyARM/kernel

還要有fa_codec_ctrl這檔案，來控制音效開關，複製至板子並讓它可執行(chmod)
cp fa_codec_ctrl ~/root_mkfs/bin
chmod 777 /bin/fa_codec_ctrl

在板子端，輸入以下指令製造連結：
mkdir /dev/snd
ln -s /dev/pcmC0D0c /dev/snd/pcmC0D0c
ln -s /dev/pcmC0D0p /dev/snd/pcmC0D0p
ln -s /dev/controlC0 /dev/snd/controlC0
ln -s /dev/pcmC1D0c /dev/snd/pcmC1D0c
ln -s /dev/pcmC1D0p /dev/snd/pcmC1D0p
ln -s /dev/controlC1 /dev/snd/controlC1
ln -s /dev/seq /dev/snd/seq
ln -s /dev/timer /dev/snd/timer
之後執行
fa_codec_ctrl tiny4412
耳機插綠孔，執行./madplay xxx.mp3，即可聽到音樂。
