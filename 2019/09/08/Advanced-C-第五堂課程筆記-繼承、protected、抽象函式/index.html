CH 12.2 類別的繼承

Rectangle跟Circle類別的物件都有Draw函式，可以將Rectangle跟Circle類別的物件放在同一個陣列裡嗎?
新增shape類別來表達不同形狀的共通點，但是以下會編譯錯誤:

讓Rectangle類別繼承shape類別

public Shape表示公開繼承，繼承B類型後，A類型就可「隱性轉型」成B類別。以上例來說，長方形跟圓形可隱性轉型成Shape，所以vector<Shape> shapes = {r, c}; 會過。反過來說，長方形跟圓形是Shape的衍生(derived)類別，Shape是長方形跟圓形的基底(base)類別。衍生(derived)類別物件就可「隱性轉型」成基底(base)類別物件。

CH 12.3 透過基底類別呼叫衍生類別函式

既然長方形是一種形狀(Shape)，那可以對Shape類別物件呼叫Draw函式嗎?

以上會編譯錯誤。解決方法是在Shape內新增成員函式Draw() const，當然也需要有函式定義。

但為什麼不是呼叫Rectangle裡面的Draw?
為什麼衍生可以隱性變基底? 這是因為每個衍生物件都會包含一個基底類別物件，透過繼承發生的隱性轉型不會產生新的物件，而是「取得」基底類別部分的子物件。
以上例來說，Rectangle物件內會有兩個Draw函式。

s是r裡Shape類別物件的複製品。s.Draw();呼叫的是Shape::Draw() const，這是因為s物件內並沒有Rectangle::Draw() const。
不過基底類別物件可參考衍生類別物件，所以可以把Shape s = r改成
const Shape& s = r;但即使這樣，s依然只能呼叫Shape裡的Draw。

在Shape class內的void Draw() const;改成virtual void Draw() const;，C++為了效率考量，需要在成員函式前加上virtual修飾字才會幫你試著呼叫衍生類別內的同名成員函式。

以上加上virtual修飾字，才真的呼叫了長方形內的Draw:
[Rectangle] x: 2, y: 3, width: 2.5, height: 4.5

加上virtual修飾字—虛擬函式，才能被衍生類別所overriden(覆載)

CH 12.4 使用基底類別為元素的容器

將兩個衍生類別存放在基底類別物件的陣列裡。


vector<Shape> shapes = {r, c};
上面語句是複製一份基底類別物件的內容到陣列裡，失去衍生類別物件的內容。

印的依舊是Shape裡的Draw。想讓他們分別印方型跟圓形，那可以改成存參考嗎? 但是沒有參考的陣列。
我們可以把
vector<Shape> shapes = {r, c};改成在陣列內存指標
vector<const Shape*> shapes = {&r, &c};
