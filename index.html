要是板子作業系統損壞，該如何復原

首先要製作一張還原卡：
將SD卡插入電腦，並確認它的代號(使用linux命令df或fdisk -l)，應該是sdb接某數字。

再來，下載課程提供的sd_fusing.tar.bz2壓縮檔，並解壓縮
tar -xvf sd_fusing.tar.bz2
再執行以下指令：
cd sd_fusing
sudo sh ./sd_fusing.sh /dev/sdb

製作完還原卡，就是重灌。在關機的情況下，插入製作好的sd卡，把開關對面的黑桿往下搬，即可設定成sd卡開機，開機後執行以下指令：
(板子端) In Exynos4412 bootloader stage
1 emmc open 1
2 dnw
4 mmc write 1 0xc0000000 0 0x10
5 dnw
7 mmc write 1 0xc0000000 0x10 0x1C
8 dnw
10 mmc write 1 0xc0000000 0x30 0x21D
11 emmc close 1
(電腦端)注意現在目錄在init_boot下，其中包含E4412_N.bl1.bin, bl2.bin,u-boot.bin跟zImage
3 sudo dnw E4412_N.bl1.bin

fbv需要libpng-1.4.1跟libungif-4.1.4先編譯完成，兩者編譯過程相同，如下
先解壓縮後，執行
./configure --host=arm-linux CC=arm-linux-gcc --prefix=/usr/local/arm/4.5.1/arm-none-linux-gnueabi/
這是因為執行configuration後會自動生成Makefile，但是裡面參數可能是for x86，因此需要
--host=arm-linux把要執行的平台換成arm
CC=arm-linux-gcc 以arm平台的編譯器編譯
--prefix=/usr/local/arm/4.5.1/arm-none-linux-gnueabi/ 告訴configuration要去那找library

之後就會生成Makefile檔，輸入以下指令
make -j4 make會4倍快
sudo -s 如果直接make install會有些指令找不到，所以先打前述指令sudo -s

原來Makefike的
LIBS = -ljpeg
需改成
LIBS = -ljpeg -lpng -lungif
再次make即可編譯成功

CH 12.2 類別的繼承

Rectangle跟Circle類別的物件都有Draw函式，可以將Rectangle跟Circle類別的物件放在同一個陣列裡嗎?
新增shape類別來表達不同形狀的共通點，但是以下會編譯錯誤:

讓Rectangle類別繼承shape類別

public Shape表示公開繼承，繼承B類型後，A類型就可「隱性轉型」成B類別。以上例來說，長方形跟圓形可隱性轉型成Shape，所以vector<Shape> shapes = {r, c}; 會過。反過來說，長方形跟圓形是Shape的衍生(derived)類別，Shape是長方形跟圓形的基底(base)類別。衍生(derived)類別物件就可「隱性轉型」成基底(base)類別物件。

CH 12.3 透過基底類別呼叫衍生類別函式

既然長方形是一種形狀(Shape)，那可以對Shape類別物件呼叫Draw函式嗎?

以上會編譯錯誤。解決方法是在Shape內新增成員函式Draw() const，當然也需要有函式定義。

但為什麼不是呼叫Rectangle裡面的Draw?
為什麼衍生可以隱性變基底? 這是因為每個衍生物件都會包含一個基底類別物件，透過繼承發生的隱性轉型不會產生新的物件，而是「取得」基底類別部分的子物件。
以上例來說，Rectangle物件內會有兩個Draw函式。

s是r裡Shape類別物件的複製品。s.Draw();呼叫的是Shape::Draw() const，這是因為s物件內並沒有Rectangle::Draw() const。
不過基底類別物件可參考衍生類別物件，所以可以把Shape s = r改成
const Shape& s = r;但即使這樣，s依然只能呼叫Shape裡的Draw。

在Shape class內的void Draw() const;改成virtual void Draw() const;，C++為了效率考量，需要在成員函式前加上virtual修飾字才會幫你試著呼叫衍生類別內的同名成員函式。

以上加上virtual修飾字，才真的呼叫了長方形內的Draw:
[Rectangle] x: 2, y: 3, width: 2.5, height: 4.5

加上virtual修飾字—虛擬函式，才能被衍生類別所overriden(覆載)
